# Workspace Launch 

NDP provides flexible options for launching and running workspaces, allowing users to choose the environment that best matches their research needs and available resources. Current capabilities include:

### NDP JupyterHub (Default Service)

Hosted by the National Research Platform (NRP) this option allows users to launch interactive Jupyter notebooks directly within the NDP ecosystem. Through JupyterHub, you can develop workflows, share projects with collaborators, and store your work persistently. Learn more about how to get started on the [JupyterHub page](../workspace-launch/jupyterhub.md).

### AWS Integration

Users with access to AWS credits can deploy their NDP workspaces on Amazon Web Services, leveraging scalable cloud infrastructure while still using the NDP software stack. This option is ideal for projects requiring flexible compute or large-scale experiments. For guidance on setting up your own NDP JupyterHub on AWS, please [contact the NDP team](../contact.md).


### NDP Endpoints (On-Premise Integration)

Institutions or groups with local computing resources can connect them to NDP through endpoints. This enables the use of NDP’s catalog, data management tools, and workspace features while retaining control over on-premise infrastructure. Learn more about endpoint setup on the [Endpoints page]().
.

**Note**: During the current developmental phase, compute resources are primarily available through NDP JupyterHub. If you are interested in enabling AWS integration or setting up on-premise endpoints, please [reach out](../contact.md) to the NDP team for support.
